# Weather Data Pipeline

**ELT-пайплайн** (Extract-Load-Transform) для автоматизованого збору, обробки та аналітики погодних даних.

Проєкт реалізовано за допомогою двох послідовних модулів на Python:

1. **Збір та збереження (Extract & Load):** Перший скрипт зчитує сирі дані через API та без змін імпортує їх у буферну схему `raw` (Staging-шар) бази даних PostgreSQL.
2. **Трансформація та завантаження (Transform & Load):** Другий скрипт витягує нові дані з шару `raw`, очищує, валідує та трансформує їх, після чого записує у кінцеву аналітичну схему `public`. Для інкрементальної обробки скрипт звертається до схеми `metadata`, де фіксує дату останніх опрацьованих даних — це дозволяє розрізняти вже оброблені записи від нових.

*Пайплайн повністю підготовлено для автоматизації та запуску за розкладом за допомогою Airflow та Docker Compose.*

## Технологічний стек та Архітектура

### Технології:
* **Мова програмування:** Python 3.x
* **Оркестрація:** Apache Airflow
* **СУБД:** PostgreSQL
* **Контейнеризація:** Docker & Docker Compose
* **Контроль версій:** Git

### Схема баз даних:
* **`raw`** — буферна зона (Staging), куди завантажуються первинні сирі дані з API.
* **`metadata`** — технічна схема, яка зберігає відмітки часу (timestamps) останньої успішної обробки для збереження інкрементальності.
* **`public`** — фінальна аналітична схема з очищеними та трансформованими даними, готовими до аналізу чи візуалізації.
